# Checklist de Publicacao

## O que voce precisa fazer

1. Criar projeto no Supabase.
2. Criar repositorio no GitHub.
3. Criar/importar projeto na Vercel conectado ao GitHub.
4. Configurar variaveis de ambiente na Vercel.
5. Criar o primeiro usuario no Supabase Auth e executar o bootstrap de desenvolvedor.

## O que eu consigo fazer daqui

1. Preparar migrations e schema.
2. Preparar app Next.js.
3. Preparar estrutura para Vercel.
4. Criar/ajustar arquivos antes do commit.
5. Quando houver acesso GitHub/Vercel/Supabase pelo conector, posso publicar e verificar o deploy.

## Ordem recomendada

1. Validar build local.
2. Subir para GitHub.
3. Importar na Vercel.
4. Aplicar migrations Supabase.
5. Configurar env vars na Vercel.
6. Fazer preview.
7. Testar login, cadastro pendente, aprovacao e dashboard.
8. Promover para producao.
